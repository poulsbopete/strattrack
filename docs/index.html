For Elastic Solution Architects

Turn opportunity chaos into timely, AI-assisted wins

A short pitch on why a local Elasticsearch notebook plus Claude Desktop MCP helps you live in Salesforce—with better context, faster updates, and clearer account plans—so you can close more opportunities.

Open the StratTrack repo

The SA job is a firehose

  • Dozens of opportunities across stages, regions, and solution areas.
  • Every meeting produces nuance—technical blockers, champions, procurement, Elastic Security vs Observability positioning. Granola is a natural place to capture it; many teams drop those notes into Google Drive (a shared, manager-visible folder) so the narrative isn’t trapped on one laptop.
  • Claude Desktop chat can correlate Drive files with Salesforce opportunities and account planning—especially when you also index takeaways in StratTrack (elastic_add_note / elastic_search_opp) so the model can search durable history, not just the current thread.

IT / naming / automation pattern: Granola → Drive → AI / Salesforce.

Salesforce is necessary—not sufficient

  • Opportunity fields are great for stage, amount, close date—not for rich running narrative.
  • AI assistants in the chat window forget tomorrow what you typed today unless you re-feed context.
  • Account planning needs continuity: last week’s discovery, yesterday’s architecture sketch, today’s exec ask.

StratTrack: your durable SA memory

StratTrack stores structured notes in Elasticsearch on your machine (Docker/Podman), wired to Claude Desktop via MCP tools: elastic_add_note, elastic_search_opp, elastic_get_1_2_3.

  • Searchable by account, opportunity, stage, title, content, and tags you store (e.g. blockers).
  • Yours—local index, optional auth—built for SA workflow, not a generic cloud scratchpad.

Why Elasticsearch for this?

  • Instant retrieval when you ask the model to “pull everything on Acme renewal blockers.”
  • Structured + text: tie notes to opportunity names, stages, and blocker tags for better answers.
  • Completion history: what the model produced stays indexed—audit trail beyond a single chat.

AI that stays on time—not just on brand

  • Before a call: search the account; draft talking points from recent notes and 1–2–3 themes.
  • After a call: append a crisp elastic_add_note with decisions, risks, and next steps.
  • Fewer tokens: the model searches the index instead of you pasting ten pages of history every time.

Account planning that compounds

  • Build a living map of stakeholders, use cases, and competitive landmines per account.
  • Use elastic_get_1_2_3-style summaries for weekly rhythm: what moved, what’s blocked, what’s next.
  • Align narrative with Salesforce opportunity reality—same names, same deals, less drift.

Close more opportunities

  • Faster follow-through when next steps live in one searchable place.
  • Fewer dropped threads when blockers and IT/security notes resurface automatically in search.
  • Better coaching handoffs—your manager or pod can reason over the same indexed context (future-friendly).

Getting started (high level)

  1. Clone the repo (or download ZIP)—scripts need files on disk.
  2. Run ./scripts/quickstart-local-es.sh for Elasticsearch + index (or build + init scripts separately).
  3. Install the MCP extension from GitHub Releases (strattrack-elasticsearch.mcpb).
  4. Use Claude Desktop to add notes and search before big moments.
